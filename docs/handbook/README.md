# docs/handbook — продуктовая документация ihichat

Источники двух распространяемых PDF (обе — на русском):

| Файл | Документ | Аудитория |
|---|---|---|
| `user-guide.html` | «ihichat — руководство пользователя» | все сотрудники (не-ИТ, максимально просто) |
| `tech-doc.html` | «ihichat — техническое описание» | ИТ: эксплуатация + архитектура |

HTML — единственный источник правды; PDF и скриншоты — сборочные артефакты
(`dist/`, `shots/`, гитигнорятся). Первая версия: 22.07.2026.

## Правило актуализации

**PR, который меняет видимое пользователю поведение, админку или процедуры
эксплуатации/деплоя, обновляет соответствующий раздел handbook** (обычно —
несколько строк в одном из HTML; скриншоты пересъёмка только если изменился
экран). Это часть Definition of Done наряду с CLAUDE.md. Правило дешёвое:
документ разбит на нумерованные секции, правка локальна.

Когда пересобирать PDF целиком: перед раздачей людям (онбординг новой волны,
передача ИТ), после заметной пачки правок, при смене версии в шапке документа
(поднимите «Версия документа» и дату в обоих HTML).

## Пересборка с нуля (скриншоты + PDF)

Требуется: локальный дев-сервер на **:4001** (`PORT=4001 mix phx.server`),
дев-Postgres, Playwright из `~/node_modules` (как у test/e2e).

```sh
# 1. Демо-данные (идемпотентно): русские персонажи, DM/группа/канал/треды,
#    фото, реакции, папка, инвайт, super_admin irina с TOTP.
mix run docs/handbook/tools/seed_demo.exs

# 2. Скриншоты всех экранов (логин, чаты, меню, треды, настройки, админка, мобильные).
NODE_PATH=~/node_modules node docs/handbook/tools/shots.js

# 3. Кропы (экраны входа, админка).
mix run --no-start docs/handbook/tools/postprocess.exs

# 4. ГЛАЗАМИ проверить shots/16-admin.png и shots/21-new-chat.png:
#    туда попадают реальные аккаунты дев-базы — размыть хелпером в postprocess.exs.

# 5. PDF → dist/
NODE_PATH=~/node_modules node docs/handbook/tools/topdf.js
```

Если правился только текст — достаточно шага 5.

## Грабли (уже собранные)

- Селекторы UI капризные — в `tools/shots.js` зафиксированы рабочие
  (контекстное меню/тред/эмодзи/«Новая беседа»); комментарий в шапке скрипта.
- `/admin` требует включённого TOTP у админа: сид включает его Ирине и кладёт
  секрет в `.demo.json`; `shots.js` сам считает текущий код через `mix run -e`.
- Сайдбар с бейджами непрочитанного снимается ДО открытия чатов.
- Локаль интерфейса берётся из Accept-Language → в Playwright задан
  `locale: "ru-RU"`.
- Демо-пользователи в дев-базе: anna/boris/maria/dmitry/irina, пароль
  `demo-pass-1234` (в проде их нет и не должно быть).
