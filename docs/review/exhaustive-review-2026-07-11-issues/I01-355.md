# Устранить краш рендера ChatLive на пустом авто-названии группы и закрыть пробелы empty-state/`initials`

**Приоритет:** P0 — `initials("")` роняет рендер всего `ChatLive` (сайдбар стримит ВСЕ диалоги пользователя, одна такая группа блокирует открытие мессенджера целиком, восстановление из UI невозможно) — по рубрике «app crash = P0».
**Лейблы:** `bug` `P0: warning!` `backend` `UI/UX` `ready-for-agent`
**Находки ревью:** R001, R060, R079, R201

## Контекст

Все находки живут в рендер-слое личного мессенджера (`EdenWeb.ChatLive`) и его хелперах идентичности/пустоты, плюс дублирующий `initials/1` в `SettingsLive`. Общий корень — **хрупкость хелперов вокруг «имени» и «пустоты»**: `initials/1` не защищён от пустой строки, авто-название группы `title/2` может стать `""`, а empty-state отрисован только для комнат. Это даёт один реальный P0-краш (R001), один UX-провал (R060), продублированную хрупкость (R201) и отсутствие тестов на оба граничных состояния (R079). Логично чинить их одним заходом: фикс `initials` и `title` закрывает краш, empty-state закрывает UX, тесты фиксируют оба.

## Находки и доказательства

### R001 · P0/bug · `lib/eden_web/live/chat_live.ex:13131` — `initials("")` падает на пустом авто-названии группы

- **Проблема:** `initials/1` не защищён от пустой строки, а `title/2` для авто-названной группы (без явного `title`) возвращает `""`, когда у группы не осталось других активных участников.
- **Доказательство в коде:** хелпер без гарда:
  ```elixir
  # chat_live.ex:13131
  defp initials(name), do: name |> String.first() |> String.upcase()
  ```
  `String.first("")` возвращает `nil`, а `String.upcase(nil)` поднимает исключение (`ArgumentError`/`FunctionClauseError`).

  Пустая строка приходит из `title/2` для авто-названной группы:
  ```elixir
  # chat_live.ex:12981
  defp title(%{is_group: true} = conversation, user) do
    conversation
    |> active_members()                              # 13104: filter is_nil(left_at)
    |> Enum.reject(&(&1.user_id == user.id))
    |> Enum.map_join(", ", & &1.user.display_name)   # [] → ""
  end
  ```
  Путь к пустому списку реален: `Eden.Chat.remove_group_member/3` (`chat.ex:462`) вызывает `leave_and_maybe_gc/2` (`chat.ex:401`), который ставит `left_at` **удалённому**, а GC (`all_members_left?/1`, `chat.ex:414`) срабатывает только когда НЕТ активных участников. Владелец-remover сам остаётся активным (`left_at: nil`) — это явно закомментировано:
  ```elixir
  # chat.ex:474-476
  # The remover always stays a member, so the conversation isn't GC'd here — the
  # "removed" notice always lands. ...
  ```
  Поэтому группа, из которой владелец удалил всех, кроме себя, НЕ собирается GC и продолжает жить в списке. `list_conversations` джойнит по membership владельца (`is_nil left_at`) и стримит эту группу в сайдбар. Два колл-сайта роняют рендер, когда у группы нет `avatar_key` (тогда `src == nil` → рисуются инициалы):
  ```elixir
  # chat_live.ex:8745 — conversation_item → .avatar name={title(...)} → initials(@name) на 8709
  <.avatar name={title(@conversation, @user)} src={conversation_avatar_src(@conversation, @user)} .../>
  ```
  ```elixir
  # chat_live.ex:11793 — conv_profile_panel (панель редактирования группы)
  <span :if={!entry && !@conversation.avatar_key}>
    {initials(title(@conversation, @user))}
  </span>
  ```
  Подтверждение из верификатора (confidence=medium, status=verified): приоритет поднят P1→P0, т.к. краш блокирует открытие мессенджера, а восстановиться из UI нельзя.
- **Влияние:** сайдбар стримит ВСЕ диалоги пользователя. Единственная авто-названная группа-«соло» роняет рендер всего `ChatLive` — пользователь не может открыть мессенджер вообще, пока состояние не изменится извне. Это краш, а не косметика.

### R060 · P2/ux · `lib/eden_web/live/chat_live.ex:3525` — нет empty-state для пустого DM/группового диалога

- **Проблема:** empty-state (`#messages-empty` с медальоном) рендерится ТОЛЬКО для комнат (гейт `:if={@selected.channel_id}`). Свежесозданный 1:1 или группа (через «New conversation» диалог создаётся до первого сообщения) открывается с пустым `#messages` и голой пустой панелью — ни приветствия, ни подсказки. Комментарий рядом («DMs / threads carry their own placeholders») вводит в заблуждение: для DM/группы плейсхолдера в шаблоне НЕТ (grep по `No messages`/`ed-room-empty`/`placeholder` не находит DM-ветки).
- **Доказательство в коде:**
  ```elixir
  # chat_live.ex:3519-3533
  <%!-- Room empty-state (#154), shown while #messages holds no streamed rows. ...
        Rooms only (DMs / threads carry their own placeholders). --%>
  <div :if={@selected.channel_id} id="messages-empty" role="status" class="ed-room-empty">
    <div class="ed-room-empty__medallion" aria-hidden="true">
      <.icon name="hero-chat-bubble-left-right" class="size-7" />
    </div>
    <p class="ed-room-empty__title">{gettext("No messages yet")}</p>
    <p class="ed-room-empty__sub">
      {gettext("Be the first to post in #%{room}.", room: @selected.name)}
    </p>
  </div>
  ```
  Reveal через CSS-сиблинг `:has()` (не in-container, чтобы не ломать якорение стрима):
  ```css
  /* app.css:2901 */
  #messages:not(:has([id^="messages-"])) + .ed-room-empty { display: block; }
  ```
  Аналог для DM/группы отсутствует — status=advisory, confidence=high (не верифицировано вторым проходом, но подтверждено чтением: DM-ветки empty-state в шаблоне нет).
- **Влияние:** пустая белая область в только что открытом чате выглядит как «сломалось / не загрузилось». Комнаты получают аккуратный empty-state, личные диалоги — нет; несогласованно и портит первое впечатление от нового чата.

### R079 · P2/test-gap · `lib/eden_web/live/chat_live_test.exs` (рендер title/initials + пустой DM) — нет тестов на краевые состояния

- **Проблема:** нет теста на (1) рендер авто-названной группы, где не осталось других активных участников (`title == ""` → `initials("")` — краш из R001); (2) открытие пустого DM/группы (наличие/отсутствие empty-state в `#messages`).
- **Доказательство в коде:** `test/eden/chat_test.exs` покрывает `remove_group_member/3` (строки 2357–2544) и GC последнего участника (2240) — но всегда на **титулованных** группах (`title: "Crew"`), поэтому пустой `title`/`initials` там не проявляется. `test/eden_web/live/chat_live_test.exs` покрывает роли группы (`describe` со строки 54), typing, реакции и #326-краш («opening a group with another member's text message doesn't crash the render», строка 702) — но НЕ рендер соло-группы с `title == ""` и НЕ empty-state пустого DM. Album-раскладка тестируется исчерпывающе, а граничные состояния идентичности/пустоты — нет. status=advisory, confidence=high.
- **Влияние:** именно эти невыраженные состояния дают P0-краш (R001) и UX-провал (R060); без тестов регрессия на `title`/`initials` или на empty-state пройдёт незамеченной.

### R201 · P3/quality · `lib/eden_web/live/settings_live.ex:1628` — `initials/1` без защиты от nil/пустого (несогласовано с `AdminLive`)

- **Проблема:** `SettingsLive.initials/1` вызывает `String.first() |> String.upcase()` без гарда, идентично баговому `ChatLive.initials/1`. Сейчас `display_name` обязателен (`validate_required`, min 1), поэтому на практике не пуст, но защита отсутствует, а в `AdminLive` аналогичный хелпер защищён.
- **Доказательство в коде:**
  ```elixir
  # settings_live.ex:1628 (незащищённый)
  defp initials(name), do: name |> String.first() |> String.upcase()
  ```
  ```elixir
  # admin_live.ex:734-737 (защищённый — эталон)
  defp initials(name) when is_binary(name) and name != "",
    do: name |> String.first() |> String.upcase()
  defp initials(_), do: "?"
  ```
  Помимо этого, почти-одинаковые хелперы `initials`/`me_initials`/`channel_initials`/`avatar_src` продублированы минимум в 4 модулях: `ChatLive`, `SettingsLive`, `AdminLive` (`admin_live.ex:729-737`) и `ShellComponents` (`me_initials/1` на `shell_components.ex:409-412`, `channel_initials/1` на 423-429; последний через `String.split(trim: true)` + `map_join` уже безопасен к `""`). status=advisory, confidence=medium.
- **Влияние:** хрупкость — если инвариант «`display_name` всегда непустой» ослабнет (или в рендер прилетит частично заполненный struct), профиль в Settings упадёт. Плюс дублирование усложняет сопровождение.

## Критерии приёмки

- Рендер `ChatLive` (сайдбар + панель группы) не падает при авто-названной группе без других активных участников; в аватаре — `"?"`, в имени — непустой фолбэк.
- Пустой 1:1/групповой диалог показывает аккуратный empty-state (согласованный с комнатным), reveal работает через CSS-сиблинг, якорение стрима не сломано (сообщения по-прежнему добавляются снизу, без «появляется только после refresh»).
- `SettingsLive.initials/1` защищён так же, как `AdminLive`.
- Новые LiveView-тесты покрывают соло-группу и empty-state; `mix check` зелёный (format / warnings-as-errors / credo --strict / sobelow / deps.audit / test), `mix dialyzer` зелёный; регрессий в существующих group/room-тестах нет.

## Заметки

- **Развилка (шаг 5):** вынос общего `initials`/`avatar_src` в shared-модуль — рефактор через 4 модуля; полезен для сопровождения, но не обязателен для закрытия P0. Можно оставить P3-хвостом отдельным PR, если хочется держать P0-фикс минимальным и легко-ревьюабельным.
- **Копирайт empty-state (шаг 3)** — продуктовое микрорешение (точные строки для DM vs группы); дефолт предложен, финал за человеком/дизайном, но не блокирует агента.
- Все находки подтверждены чтением актуального кода; номера строк совпали с входом (`initials` 13131, `title` 12981, empty-state 3525, settings 1628). R001 — единственная verified (medium); R060/R079/R201 — advisory, но подтверждены при чтении.
- `ShellComponents.me_initials/1` (`shell_components.ex:409`) уже имеет `is_binary`-гард + фолбэк `"?"`, но НЕ ловит пустую строку `""` (гард пропускает `""`); работает на текущем пользователе (`display_name` min 1), так что практического краша нет, но при выносе общего хелпера стоит закрыть и его. `channel_initials/1` уже безопасен к `""`.
