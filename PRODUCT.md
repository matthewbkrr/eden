# Product

## Register

product

## Users

A small, closed circle (~20 people at launch, built to grow) who join **only by
invite link** — no public sign-up, no email. They chat 1:1 and in groups, sending
text and photos in real time, mostly from phones on Russian networks talking to a
server hosted abroad (so the connection can be flaky). Their context is everyday
personal messaging: quick, frequent, glanceable.

## Product Purpose

eden is a private realtime messenger (text + photos) for people who already know
each other. Success = it feels as fast, calm, and trustworthy as the best
mainstream messengers, while staying invite-only and resilient on a shaky
cross-border connection. The interface should disappear into the conversation.

## Brand Personality

Calm, restrained, grown-up. Three words: **quiet, precise, trustworthy.** It
should feel like a focused tool (Linear) wearing the familiar, comfortable shape
of a chat app (Telegram) — never loud, never playful, never attention-seeking.
The product earns trust through familiarity and consistency, not novelty.

## Anti-references

- **Instagram-style bright gradients / gradient text** — no saturated gradient
  branding, no candy colors.
- **Discord / Slack visual overload** — no wall of panels, badges, and competing
  colors at once; keep one calm surface in focus.
- **Playful / cartoonish UI** — no oversized radii, emoji-as-decoration, bouncy
  motion, sticker-forward chrome.
- **Loud light UI** — the default mood is dark; light theme exists but stays
  equally restrained (no harsh pure-white glare).

## Design Principles

1. **The conversation is the product.** Chrome recedes; messages and people are
   the only things that should draw the eye.
2. **Earned familiarity over invention.** Use the affordances people already know
   from great messengers; never reinvent a standard control for flavor.
3. **Calm by default, color with intent.** One cobalt accent carries actions,
   links, and your own messages — everywhere else is quiet neutral.
4. **Three themes, one feeling.** Dark, light, and system must all read as the
   same calm, precise product — not a bright mode and a moody mode.
5. **Resilient and honest.** The UI tells the truth about connection state
   (reconnecting, sending, sent) without alarm — degrade softly.

## Accessibility & Inclusion

Target **WCAG 2.1 AA**: body text ≥4.5:1, large/secondary text ≥3:1, visible
focus rings on every interactive element, full keyboard operability. Honor
`prefers-reduced-motion` (transitions become crossfades/instant). Color is never
the only signal (online state, unread, errors also carry shape/text/icon).
Themes follow the OS via `prefers-color-scheme` when set to "system".
